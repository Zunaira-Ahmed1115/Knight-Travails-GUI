## 📌 Overview

**Knight’s Travails** is a classic graph traversal problem where a knight must find the shortest path between two squares on a chessboard.

This project solves the problem using **Breadth-First Search (BFS)** and visualizes the result through a **Java Swing GUI**, making it both educational and interactive.

---

## 🎯 Objectives

- Understand and implement **BFS (Breadth-First Search)**
- Model a chessboard as a **graph structure**

- Build a **graphical Java Swing application**
- Improve understanding of **OOP and data structures**

---

## ♟️ Problem Description

The chessboard is modeled as an `8×8 grid`, where each square is a node in a graph.

- Each legal knight move represents an edge.
- The task is to find the **minimum number of moves** from a start position to a destination.
- The knight must stay within board boundaries.

---

## 🧠 Methodology

- Chessboard implemented using a **2D JButton grid**
- BFS algorithm explores all possible moves level by level
- A **queue** stores traversal paths
- A **visited matrix** avoids revisiting nodes
- GUI allows users to select start and end positions
- Path is highlighted visually using colors

---

## ⚙️ Algorithm Used

### 🔍 Breadth-First Search (BFS)

BFS guarantees the shortest path in an unweighted graph by exploring all neighboring nodes before moving deeper.

**Key Idea:**
> Explore all possible knight moves step-by-step until the destination is reached.
